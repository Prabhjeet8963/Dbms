## Directory Organization

The repository is organized under a single dedicated database systems workspace:

### Dbms/
* mysql/: Relational databases containing schema designs, standard integrity constraints, and query suites (Company Database, Sales, Supplier Inventory, Student Athletics, and Inbuilt HR functions) grouped directly with their lab requirements.
* nosql-mongodb/: Document-oriented schema-less databases using MongoDB (Hotel Registry).
* er-diagrams/: Conceptual models (NHL League, University Registrar, and Company Organization) transcribed into Mermaid.js notation rendering natively in GitHub.
* archive/: Reference slides, PDFs, course syllabus structures, completion certificates, and old raw lab files preserved for academic records.
